# 12. 캘린더 스티커 추가 (대규모수정사항에 추가?) 후속제안 작업업

## 변경 배경
- 캘린더에서 날짜별 완료 개수를 시각적으로 표현해 사용자 동기 부여 필요
- 날짜 셀만으로는 “오늘 몇 개 완료했는지” 한눈에 파악하기 어려움
- 8번 문서 후속 제안(상세 화면 이모티콘/스티커)과 별개로, 캘린더 그리드에 완료 개수별 스티커 표시 요구 반영

## 변경 목표
- 캘린더 날짜 셀에 해당 날짜 **완료 개수(0~5+)**에 따라 스티커 이미지(done0~done5) 표시
- 할 일이 있는 날만 스티커 영역 노출, 미래/할 일 없는 날은 스티커 미표시
- 셀 레이아웃 안정화(숫자–스티커 간격, 스티커 영역 크기 고정, 이미지 비율 유지)

## 적용 범위
- 일정(캘린더) 화면 그리드 셀 UI
- 날짜별 완료 개수 집계 로직 활용(doneTodoDateMap)
- 스티커 에셋 및 스타일

## 데이터 모델 변경
- **없음.** 기존 `todos` / `isDone`, `createdAt` 기준 집계(`doneTodoDateMap`, `activeTodoDateMap`)만 사용.

## 화면/플로우 변경 사항
### 캘린더 그리드
- 각 날짜 셀: 날짜 숫자 아래 **스티커 영역** 추가
- `hasTodo`(해당 날짜에 활성 todo 1개 이상)인 날만 스티커 표시
- 완료 개수에 따른 이미지 매핑:
  - 0개 → `done0.png`
  - 1개 → `done1.png`, 2개 → `done2.png`, … 5개 이상 → `done5.png`
- 셀 최소 높이 80px, `overflow: hidden`으로 스티커가 칸 밖으로 나가지 않도록 처리
- 숫자와 스티커 사이 간격, 스티커 영역 크기 강제 고정(CSS/인라인)

## 구현 파일
- `5todolist/src/5todolist/SchedulePage.tsx` — doneTodoDateMap 집계, 셀별 stickerSrc 계산 및 `<img>` 렌더
- `5todolist/src/App.css` — `.calendar-day-number`, `.calendar-sticker-area`, `.calendar-sticker` 등 스티커 레이아웃/크기
- `5todolist/src/assets/done0.png` ~ `done5.png` — 완료 개수별 스티커 이미지

- [v] 셀 높이·스티커 영역·이미지 크기(object-fit 포함) 고정으로 레이아웃 안정화
