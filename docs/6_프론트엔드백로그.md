# 6. 프론트엔드 백로그 (PRO-F)

## 규칙
- 백로그 ID는 `PRO-F-번호` 형식으로 작성한다.
- 예: `PRO-F-1`, `PRO-F-2`, `PRO-F-3`
- 문서(PRD, SED, 결정로그)에서는 아래 ID를 그대로 참조한다.

## 백로그 항목

| ID | 제목 | 설명 | 우선순위 | 상태 |
|---|---|---|---|---|
| PRO-F-1 | 모바일 레이아웃 프레임 구축 | 모바일 웹 기준의 공통 레이아웃/여백/하단 탭 구조를 정의하고 적용한다. | High | Todo |
| PRO-F-2 | 스플래시(로딩) 화면 구현 | 로고, 서브카피, 진입 애니메이션/전환 타이밍을 포함한 첫 화면을 구현한다. | High | Todo |
| PRO-F-3 | 오늘 할 일 5개 제한 로직 | 오늘 할 일 생성 시 최대 5개까지만 등록되도록 UI/검증 로직을 구현한다. | High | Todo |
| PRO-F-4 | 미래 일정 작성 제한 | 오늘 기준 입력만 허용하고 미래 날짜 작성 경로를 제거한다. | High | Todo |
| PRO-F-5 | 할 일 상태 분류 UI | 진행 전/진행 중/진행 완료 섹션과 체크 상태 표현을 일관되게 제공한다. | Medium | Todo |
| PRO-F-6 | 미완료 항목 정리 플로우 | 미완료 항목에 대해 삭제/보관 선택 모달(또는 바텀시트)과 결과 반영을 구현한다. | High | Todo |
| PRO-F-7 | 보관함 목록 화면 구현 | 보관함 목록, 상태 구분, 빈 상태 화면을 구성한다. | Medium | Todo |
| PRO-F-8 | 보관함 접근 인증 가드 | 보관함/과거 리스트 접근 시 비로그인 사용자를 로그인 화면으로 리다이렉트한다. | High | Todo |
| PRO-F-9 | 로그인 화면 및 인증 진입점 | 카카오 SSO/일반 이메일 로그인 UI와 로그인 성공 후 복귀 동선을 제공한다. | High | Todo |
| PRO-F-10 | 타이포/브랜드 시스템 반영 | 온글잎 박다현/온글잎 김콩해 사용 기준과 폴백 전략을 문서화하고 적용한다. | Medium | Todo |
| PRO-F-11 | 접근성/터치 영역 점검 | 터치 영역, 대비, 상태 전달(색+아이콘) 기준을 충족하도록 점검한다. | Medium | Todo |
| PRO-F-12 | 빈 화면/오류 상태 디자인 | 데이터 없음, 인증 실패, 네트워크 오류 등 예외 상태 UI를 정의한다. | Medium | Todo |
