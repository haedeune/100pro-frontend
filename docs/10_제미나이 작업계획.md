# 10_제미나이 작업계획

## 작업 목표
- 백엔드 없이 프론트엔드 환경에서 동작하는 로그인 및 회원가입 기능 구현.
- 회원가입을 통해 사용자(이름, 이메일, 비밀번호)를 생성하고, 로그인 후 마이페이지에서 이를 확인할 수 있도록 구성.
- 기존의 Mock 데이터를 신규 생성될 사용자 "백프로"(이메일: test@test.com)의 데이터로 연동되도록 수정.
- 데이터를 `localStorage` 등 브라우저 저장소를 활용하여 프론트엔드 내에서 유지하도록 구현.

## 프로세스 설계

1. **저장소 설계 (Frontend LocalStorage)**
   - `users`: 회원가입한 사용자 목록 저장 (초기값: 비어있음, 가입 시 추가)
   - `currentUser`: 현재 로그인한 사용자 정보 저장
   - `mockData`: 기존에 하드코딩된 목업 데이터. 사용자 "백프로"와 연결되도록 구조 변경.

2. **회원가입 페이지 (Signup Page)**
   - 입력 폼: 이름 (Name), 이메일 (Email), 비밀번호 (Password)
   - 액션: 가입 버튼 클릭 시 `users` localStorage에 정보 저장 후 로그인 페이지로 리다이렉션.

3. **로그인 페이지 (Login Page)**
   - 입력 폼: 이메일 (Email), 비밀번호 (Password)
   - 액션: 로그인 버튼 클릭 시 `users`를 조회하여 정보 일치 여부 확인. 성공 시 `currentUser` 세팅 후 메인 페이지(또는 마이페이지)로 이동.

4. **마이페이지 (My Page)**
   - `currentUser`의 정보를 바탕으로 사용자의 이름과 이메일 표시.

5. **기존 Mock Data 연동**
   - 현재 프론트엔드에 저장된 목업 데이터들을 수정.
   - 데이터의 소유자를 "백프로"(test@test.com)로 지정하여, 해당 계정으로 로그인했을 때 기존 데이터들이 자신의 데이터인 것처럼 노출되도록 제어.
